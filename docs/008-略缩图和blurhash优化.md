# 略缩图和 BlurHash 优化

本轮对缩略图和 BlurHash 做了三项优化。

## 1. 缩略图尺寸和质量提高

- `thumbnail_width` 从 `640` 提高到 `960`
- `thumbnail_quality` 从 `90` 提高到 `92`
- 当前配置样例和本地配置已同步更新
- 更适合摄影图预览，减少树叶、砖墙、暗部边缘这类细节的压缩糊化

## 2. 缩略图亮度与色彩修复

问题原因：

- 原来的内部预览链路没有完整接入系统色彩管理
- 对带 ICC、广色域或 HDR 映射的源图，缩略图会比系统显示结果更暗、更闷

当前实现：

- macOS 下，缩略图和 BlurHash 预览图优先走 `sips --optimizeColorForSharing`
- 先由系统图像管线完成色彩空间转换、亮度映射和缩放
- 再进入项目内的 `webp` 编码和 BlurHash 计算
- 如系统预览失败，才回退到原有内部预览链路

## 3. BlurHash 质量提高

当前实现调整如下：

- 不再使用固定 `32x32`
- 改为按原图比例缩放，长边固定为 `96`
- 使用和缩略图一致的高质量缩放链路
- 先做更明显一些的高斯模糊，再生成 BlurHash
- 根据画幅比例分配更保守的 BlurHash 组件数量
- 增量复用时也会基于现有缩略图刷新 BlurHash，不需要重建整张图片

当前策略：

- 方图使用 `4 x 4`
- 横图使用 `5 x 4`
- 竖图使用 `4 x 5`

目标：

- 保留原图宽高比特征
- 提高颜色块层次
- 让颜色过渡更平滑，减少四周暗边和振铃感
- 让占位图在加载阶段更接近真实画面氛围

## 兼容策略

- 新的 BlurHash 策略会在下次 `build` 时直接刷新 `manifest.json`
- 缩略图和 AVIF 不需要因为 BlurHash 调整而重新生成
