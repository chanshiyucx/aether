# 构建优化

本轮针对构建链路做了 3 项修复。

## 1. 进度条输出收紧

- 去掉进度条左侧的已耗时显示
- 只保留进度条、当前数量和状态文案

当前样式：

```txt
[=====================>-----] 12/19 building
```

## 2. JPEG 转 AVIF 的显示问题

问题原因：

- 8-bit 源图之前统一走手写的 YCbCr 平面编码路径
- 这条路径更适合高位深源图，不适合普通 JPEG 的稳定显示

修复方案：

- `bitDepth <= 8` 的源图改为走 `ravif` 自带的 `RGB` 编码路径
- 优先保证普通 JPEG / PNG / WebP 转 `AVIF` 后的显示正确

## 3. WebP 缩略图纯黑

问题原因：

- 高位深 `HIF / HEIF / HEIC` 解码后是 `10-bit / 12-bit / 16-bit` 数据
- 之前直接转成 `8-bit` 预览图，未按真实位深做归一化
- 对 10-bit 数据会被错误压到接近 0，导致缩略图接近纯黑

修复方案：

- 缩略图和 BlurHash 都先走单独的预览图链路
- 预览图会按源图真实位深归一化到 `8-bit`
- 再做 `fast_image_resize` 缩放和 `webp` 编码

## 兼容策略

- 本轮修复会提升内部 `pipelineVersion`
- 旧的 `AVIF` 和缩略图会在下次 `build` 时自动失效并重建
