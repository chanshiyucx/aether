# 项目初始化实施过程

## 目标

基于当前已确认的架构、命令行和数据格式文档，使用 Rust 官方 `cargo` 脚手架初始化 `aether` 项目，并补齐最小可运行的 CLI 骨架。

## 实施步骤

### 1. 使用官方脚手架初始化项目

在项目根目录执行：

```bash
cargo init --bin --name aether --vcs none .
```

初始化结果：

- 生成 `Cargo.toml`
- 生成 `src/main.rs`
- 保留现有 `docs/` 文档目录

### 2. 补充依赖

根据当前文档约束，补充了以下基础依赖：

- `clap`
- `anyhow`
- `serde`
- `serde_json`
- `toml`
- `tracing`
- `tracing-subscriber`
- `walkdir`
- `image`
- `blurhash`
- `time`

### 3. 建立最小代码结构

新增模块：

- `src/cli.rs`
- `src/config.rs`
- `src/build.rs`

职责划分：

- `cli.rs`
  定义 `aether build`
- `config.rs`
  读取 `aether.toml`
- `build.rs`
  扫描原图、生成缩略图、写入 `manifest.json` 和 `state.json`

### 4. 固化项目默认配置

新增：

- `aether.toml.example`

默认配置包含：

- `path`
- `originals_dir`
- `thumbnails_dir`
- `thumbnail_width`
- `thumbnail_format`
- `thumbnail_quality`
- `enable_blurhash`

### 5. 固化本地产物约定

新增：

- `.gitignore`

忽略内容：

- `target/`
- `.thumbnails/`
- `manifest.json`
- `state.json`
- `aether.toml`

## 当前初始化结果

当前项目已经具备：

- 官方 `cargo` 二进制项目结构
- 可执行的 `aether build`
- 本地配置样例
- 基础目录扫描能力
- 缩略图输出能力
- `manifest.json` 生成能力
- `state.json` 生成能力
- BlurHash 生成能力

## 当前实现边界

当前初始化版本只覆盖最小可运行能力，尚未扩展：

- EXIF 解析
- GPS 信息提取
- 相机参数提取
- HEIF / HIF 支持
- 外部同步脚本集成

## 运行方式

```bash
cp aether.toml.example aether.toml
cargo run -- build
```

或编译后运行：

```bash
cargo build --release
./target/release/aether build
```
