# 元数据提取

当前版本的元数据提取目标，是把摄影展示真正有消费价值的信息写入 `manifest.json`，同时保持构建流程稳定、可增量。

## 提取范围

### 1. 拍摄时间

- `DateTimeOriginal`
- `SubSecTimeOriginal`
- `OffsetTimeOriginal`

回退顺序：

1. `DateTimeOriginal`
2. `DateTimeDigitized`
3. `DateTime`

输出字段：

- `takenAt`

输出格式：

- 优先输出带时区和小数秒的 ISO 8601 风格字符串
- 如果 EXIF 中没有时区，则只输出本地时间部分

### 2. 地理位置

- `GPSLatitude`
- `GPSLatitudeRef`
- `GPSLongitude`
- `GPSLongitudeRef`
- `GPSAltitude`
- `GPSAltitudeRef`

输出字段：

- `location.lat`
- `location.lng`
- `location.alt`

当前不做：

- `country`
- `city`

原因是这两个字段需要逆地理编码，不属于本地 EXIF 提取范围。

### 3. 相机与镜头信息

- `Make`
- `Model`
- `LensModel`
- `LensMake`
- `FocalLength`
- `FocalLengthIn35mmFilm`
- `FNumber`
- `MaxApertureValue`
- `LensSpecification`
- `ExposureTime`
- `PhotographicSensitivity`
- `ISOSpeed`
- `ExposureProgram`
- `ExposureMode`
- `MeteringMode`
- `WhiteBalance`
- `Flash`
- `LightSource`
- `SceneCaptureType`
- `BrightnessValue`
- `SensingMethod`

输出字段：

- `camera.make`
- `camera.model`
- `camera.lens`
- `camera.focalLengthMm`
- `camera.focalLengthIn35mm`
- `camera.aperture`
- `camera.maxAperture`
- `camera.shutter`
- `camera.iso`
- `camera.exposureProgram`
- `camera.exposureMode`
- `camera.meteringMode`
- `camera.whiteBalance`
- `camera.flash`
- `camera.lightSource`
- `camera.sceneCaptureType`
- `camera.brightnessEv`
- `camera.sensingMethod`

规则：

- 相机信息按字段级别尽力提取
- 某个字段缺失时，不影响其余字段写入
- `maxAperture` 优先取标准 `MaxApertureValue`，缺失时回退 `LensSpecification`
- `exposureMode` 归一为短值：`auto`、`manual`、`bracket`
- `whiteBalance` 归一为短值：`auto`、`manual`
- `flash` 归一为短值：`off`、`on`、`auto`、`auto-fired`、`red-eye`、`unsupported`
- 不做品牌私有 MakerNote 翻译

### 4. 图像信息

- `Orientation`
- `ColorSpace`
- 源文件 `bitDepth` 探测结果

输出字段：

- `image.orientation`
- `image.sourceOrientation`
- `image.colorSpace`
- `image.hasHdr`
- `image.isLivePhoto`
- `image.bitDepth`

规则：

- 处理阶段会先对源图应用 EXIF Orientation，再输出 `AVIF` 和缩略图
- `image.orientation` 当前描述输出图像，因此固定为 `1`
- 如源文件带 EXIF 方向信息，会额外写入 `image.sourceOrientation`
- `colorSpace` 只表示色彩空间，不混入 HDR 判断
- `hasHdr` 当前按 `bitDepth > 8` 判断
- `isLivePhoto` 当前固定为 `false`

## 实现策略

- 所有元数据都从源文件读取，不从转码后的 `AVIF` 或缩略图反推
- 即使图片命中增量复用，仍然会刷新一次 `manifest.json` 中的元数据
- 元数据提取失败不会中断整张图片构建，只会退化为缺省字段

## 当前技术方案

- EXIF 读取：`kamadak-exif`
- HEIF/HIF 位深探测：`libheif-rs`
- 非 HEIF 图片位深探测：`image`

## 当前边界

- 只处理本地可直接提取的信息
- 只处理标准 EXIF，不解析厂商 MakerNote
- 不接入逆地理编码
- 不接入 Live Photo 配对识别
- 不做人脸、场景、关键词等推断型元数据
