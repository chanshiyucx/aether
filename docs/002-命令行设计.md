# 命令行设计

二进制名称统一为 `aether`，当前项目的 CLI 只负责一件事：构建图片产物。

> 同步到对象存储不属于 CLI 范围，由独立的 `rclone` 脚本处理。

## 设计原则

- CLI 只负责本地图片处理
- 输入源和发布目录分离
- 默认增量构建
- 是否参与构建由 macOS Finder Tag 控制
- 发布原图统一输出为 `AVIF`

## 命令

### `aether build`

```bash
aether build
```

用途：

- 扫描 `source_dir`
- 按 `source_tag` 过滤图片
- 将入选图片统一转码为 `AVIF`
- 生成缩略图
- 生成 BlurHash
- 生成 `manifest.json`
- 更新 `state.json`
- 自动删除已失效的派生产物

## 输入与输出

### 输入

- `source_dir`
- `aether.toml`
- `state.json`

### 输出

- `originals/`
- `.thumbnails/`
- `manifest.json`
- `state.json`

## 行为约定

- 默认执行增量构建
- 只处理带 `source_tag` 的图片
- 入选图片统一输出为 `AVIF`
- 源文件新增时，自动生成对应产物
- 源文件变化时，自动重新生成对应产物
- 源文件删除或移除 Tag 时，自动删除对应发布原图、缩略图和 Manifest 条目
- 构建完成后，本地产物目录应完全反映当前 `source_dir + source_tag` 状态

## 配置建议

CLI 只保留图片处理相关配置项：

- 工作目录
- 素材目录
- Finder Tag
- 发布原图目录
- 缩略图目录
- 缩略图尺寸
- 缩略图格式
- 缩略图质量
- BlurHash 开关

示例：

```toml
sourcePath = "/absolute/path/to/photos"
targetPath = "/absolute/path/to/output"
originals_dir = "gallery"
thumbnails_dir = "thumbnails"
thumbnail_width = 640
thumbnail_format = "webp"
thumbnail_quality = 82
avif_quality = 95
enable_blurhash = true
```

如需按 Finder Tag 过滤，可额外设置：

```toml
sourceTag = "Aether"
```

## 典型使用方式

### 首次构建

```bash
cargo build --release
cp aether.toml.example aether.toml
./target/release/aether build
```

### 日常增量构建

```bash
aether build
```

## 工作流

1. 准备 `aether.toml`
2. 在 `source_dir` 中给目标图片打上 Finder Tag
3. 执行 `aether build`
4. 检查 `originals/`、`.thumbnails/`、`manifest.json`、`state.json`
5. 如需上传，再单独执行外部同步脚本

## 退出码

- `0`：构建成功
- `1`：配置错误、目录缺失或运行时失败
- `2`：存在部分图片处理失败，但整体流程已完成
