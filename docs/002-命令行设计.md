# 命令行设计

二进制名称统一为 `aether`，当前项目的 CLI 只负责一件事：构建图片产物。

> 同步到对象存储不属于 CLI 范围，由独立的 `rclone` 脚本处理。

## 设计原则

- 只保留核心构建图片命令，降低理解和维护成本
- CLI 只负责本地图片处理，不接管上传、部署和远端配置
- 默认增量构建，重复执行成本可控
- 输出结构固定且清晰

## 命令

### `aether build`

```bash
aether build
```

用途：

- 扫描原图目录
- 增量生成缩略图
- 生成 BlurHash
- 生成 `manifest.json`
- 更新 `state.json`
- 自动删除已失效的派生产物

## 输入与输出

### 输入

- `originals/`
- `aether.toml`
- `state.json`

### 输出

- `.thumbnails/`
- `manifest.json`
- `state.json`

## 行为约定

- 默认执行增量构建
- 原图新增时，自动生成对应产物
- 原图变化时，自动重新生成对应产物
- 原图删除时，自动删除对应缩略图和 Manifest 条目
- 构建完成后，本地产物目录应完全反映当前原图目录状态

## 配置建议

CLI 只保留图片处理相关配置项：

- 原图目录
- 缩略图目录
- Manifest 路径
- State 路径
- 缩略图尺寸
- 缩略图格式
- 缩略图质量
- BlurHash 开关

示例：

```toml
path = "."
originals_dir = "./originals"
thumbnails_dir = "./.thumbnails"
thumbnail_width = 640
thumbnail_format = "webp"
thumbnail_quality = 82
enable_blurhash = true
```

## 典型使用方式

### 首次构建

```bash
cargo build --release
cp aether.toml.example aether.toml
./target/release/aether build
```

### 日常增量构建

```bash
aether build
```

## 工作流

1. 准备 `aether.toml`
2. 将原图放入 `originals/`
3. 执行 `aether build`
4. 检查 `.thumbnails/`、`manifest.json`、`state.json`
5. 如需上传，再单独执行外部同步脚本

## 退出码

- `0`：构建成功
- `1`：配置错误、目录缺失或运行时失败
- `2`：存在部分图片处理失败，但整体流程已完成
